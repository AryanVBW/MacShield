Features

Two shields. One extension.

Privacy blur for your chats. Web lock for your sites. Built together, works perfectly.

Privacy blur

All messages blur automatically on WhatsApp, Gmail, Discord, Slack and 10 more platforms. Hover to peek. Select to read. Nobody nearby sees a thing.

Website lock

Lock any website with a password or Touch ID. A full-screen auth wall blocks access until authenticated. No password — no entry. Works on any domain you choose.

Touch ID unlock

Enroll your Mac's Touch ID via WebAuthn. Authenticate with your fingerprint instead of typing a password. Biometric data never leaves the Secure Enclave.

Alt+X instant toggle

Hit Alt+X on any supported page to instantly toggle blur on and off. No popup needed. One keystroke to go from fully private to fully visible and back.

Blur intensity control

Slider from 2px light to 30px heavy. Three presets — Light, Medium, Deep — for quick switching. Hide mode completely removes content instead of blurring.

Auto-relock on tab leave

Enable auto-relock and the moment you switch away from a protected tab, it locks again. Return — enter password — continue. Airtight protection on shared machines.

Security & privacy

Nothing leaves your device. Ever.

Every security claim in MacShield is verifiable — the full source code is public on GitHub.

SHA-256 hashed passwords

Your master password is immediately hashed with a random 16-byte salt using the Web Crypto API. The original password is never stored anywhere in any form.

Local storage only

All data — preferences, locked sites, password hash — lives in chrome.storage.local on your own machine. No backend. No cloud. No servers. Zero outbound requests.

Secure Enclave Touch ID

Biometric authentication runs entirely inside your Mac's Secure Enclave via WebAuthn. MacShield stores only a credential reference ID — no fingerprint data, ever.

CSS-only blur

The blur feature works by applying filter:blur() CSS to page elements. It never reads, copies, logs, or processes the content of any message or email.

Session-based unlocks

Unlocked sites are tracked in chrome.storage.session which clears automatically when Chrome closes. Every new browser session starts with all sites locked again.

Fully open source

Every line of code is public at github.com/AryanVBW/MacShield under the MIT license. Audit it, fork it, verify every privacy claim independently.

FAQ

Common questions

Does MacShield read my messages? +
No. The blur is pure CSS — filter:blur() applied to page elements. MacShield never reads, copies, logs, or processes the text of any message, email, or chat. It only knows the hostname of the page you're on.
Where is my password stored? +
Your password is hashed immediately using SHA-256 with a random salt via the Web Crypto API, then the hash is stored in chrome.storage.local on your own device. The original password is never stored anywhere. Nothing is sent to any server.
Why does it need the <all_urls> permission? +
The Web Lock feature lets you lock any domain you choose — not just a hardcoded list. To inject the password lock screen before a locked page renders, the extension needs permission to run at document_start on all URLs. Without this, custom site locking would be impossible.
Does it work in incognito mode? +
Not by default. Chrome requires you to explicitly enable extensions in incognito mode. Go to chrome://extensions, find MacShield, and toggle "Allow in incognito" to enable it there.
What happens if I forget my password? +
Since passwords are stored locally as a one-way hash, there is no password recovery. You would need to remove the extension and reinstall it, which clears all settings. This is by design — no recovery means no backdoor.
Is Touch ID available on Windows? +
Touch ID is currently Mac-only via WebAuthn platform authenticator. Windows Hello support is planned for a future version. On non-Mac devices the biometric option is hidden and only password authentication is available.
